Strings in Java

Strings


Strings in java are treated as Object and as we all know for the object memory is allocated inside Heap Area. anything in double quotes are considered as a string. In string we don't have a null terminator in Java.


Types of String

  • I] Immutable Strings
  • II] Mutable Strings

* 1] Immutable String

=> It can't be changed once it created.

=> For working with immutable strings we have a inbuilt class called String.


Different ways to creating a String :- 

  1. String str="Alien";
  2. String str=new String("Alien");
  3. char[] c={'A','l','i','e','n'};
String str=new String(c);

Let's discuss the different types of creating Strings with respect to Memory area.





=> Inside Heap there are two sections when we declare any string. The first one which is Actual Heap Area and the second one is String Constant Pool(SCP). In SCP duplicates are not permitted and In the Actual Heap Area duplicates are allowed. When you create a string using new keyword then the actual memory will given inside the actual heap area and one copy of that is present inside SCP But when you create a string without using new keyword then memory is allocated inside the SCP only.



# Different Ways to Compare String

=> There are 4 methods to compare two strings in Java.


1] (==) => double equals compares reference variable.

2] equals() => equals method compare value of string.

3] equalsIgnoreCase() => string will be compared by Ignoring case sensitivity.

4] ComapreTo() => CompareTo method compare values of string lexiographically(character by character).



* 2] Mutable String

=> It can be changed once it created.

=> For working with Mutable strings we have a inbuilt class called StringBuffer and StringBuilder.


Two ways to creating a Mutable String :- 

  1. StringBuffer sb=new StringBuffer("Sachin");
StringBuilder sb1=new StringBuilder("Virat");
            
  Case1:
    String s=new String("sachin");  // 2 object (SCP and other one in heapArea)
    s.concat("tendulkar"); // 1 object(SCP)
    System.out.println(s); 
    Since String object are immutable, if we try to change the object, that change would not happen in the same memory. the changes will happen in the new memory 
    this concept we call it as "Immutable".
                                         *****VS*****
    StringBuffer sb=new StringBuffer("sachin"); // 2 objects( SCP and Other one in heapArea)
    sb.append("tendulkar"); // 1 object(SCP)
    System.out.println(sb);
Since StringBuffer is mutable, if we try to change the object, that change would happen in the same memory, this mechanism is called "mutable".

  Case2:
    String s1=new String("sachin");// 2 objects(SCP and Other one in heapArea)
    String s2=new String("sachin");// 1 Object (heapArea)
    System.out.println( s1.equals(s2) ); // equals method is implemented to check the content of String
                                        ***vs***
    StringBuffer sb1=new StringBuffer("sachin");
    StringBuffer sb2=new StringBuffer("sachin");
    System.out.println(sb1.equals(sb2); //equals method is not available in StringBuffer class,it is a part of Object class so the implementation is coming from Object class  
    which compares the reference(address) not the data.
